# Однослойный перцептрон

## linal_lab2

Реализация однослойного перцептрона **с нуля на NumPy** (без PyTorch, TensorFlow, sklearn.linear_model).

## Структура проекта

```text
├── perceptron.py    - класс Perceptron (sigmoid, forward, loss, fit, predict)
├── data.py          - подготовка данных (генерация, train/test split, стандартизация)
├── plots.py         - вспомогательные функции визуализации
├── train.py         - базовое обучение (lr=0.1, epochs=100, batch=32)
├── experiments.py   - три эксперимента (lr / batch_size / init)
├── dataset.py       - бонус 1: собственный генератор датасетов
├── losses.py        - бонус 2: Hinge loss + L2 регуляризация
├── metrics.py       - бонус 3: Precision, Recall, F1, ROC-AUC
├── momentum.py      - бонус 4: SGD с моментом
├── crossval.py      - бонус 5: 5-fold CV + grid search
└── lab2.py          - точка входа, запускает всё
```

## Бонусное задание 1 - собственный генератор

Файл `dataset.py` реализует три генератора без sklearn.

### `make_blobs()` - гауссовые облака

```python
X, y = make_blobs(
    n_samples=500,
    center0=(-2, -2),   # центр класса 0
    center1=( 2,  2),   # центр класса 1
    cov=np.eye(2),      # ковариационная матрица
    noise=0.1,          # вероятность перевернуть метку
)
```

**Почему `multivariate_normal`?**

Гауссово облако - это точки, распределённые по многомерному нормальному закону
N(μ, Σ), где μ - центр кластера, Σ - ковариационная матрица.
Именно эта функция реализует такое распределение напрямую.

Ковариационная матрица задаёт форму и ориентацию облака:

- `Σ = I` → круглое облако, одинаковый разброс по всем направлениям
- `Σ = diag(a, b)` → эллипс вдоль осей
- `Σ` с ненулевыми вне-диагональными элементами → эллипс повёрнутый

Использовать `normal` по каждой оси отдельно можно только если признаки
независимы (Σ = I). `multivariate_normal` - общий случай, корректно
учитывающий корреляцию между признаками.

### `make_circles()` - внутри / снаружи окружности

```python
X, y = make_circles(
    n_samples=500,
    r_inner=1.0,   # радиус внутреннего круга (класс 1)
    r_outer=2.5,   # внешний радиус кольца    (класс 0)
    noise=0.05,
)
```

**Почему `r = sqrt(uniform(r_min², r_max²))`?**

Интуитивно кажется: чтобы точки равномерно заполнили кольцо,
достаточно взять `r ~ Uniform(r_min, r_max)`. Но это неверно.

Площадь тонкого кольца толщиной dr на радиусе r равна 2πr dr -
она растёт с r. Если семплировать r равномерно, маленькие r
получают столько же точек, сколько большие, хотя занимают
меньшую площадь. В результате центр будет перенасыщен точками.

Чтобы точки распределились равномерно по **площади**, нужно,
чтобы плотность по r была пропорциональна r:

```text
p(r) = 2r / (r_max² - r_min²)
```

Обратная функция от CDF этого распределения даёт:

```text
r = sqrt(U · (r_max² - r_min²) + r_min²)  ,  U ~ Uniform(0, 1)
```

что эквивалентно `sqrt(uniform(r_min², r_max²))`.
Это стандартный метод обратного преобразования (inverse CDF sampling).

### Выводы о границах применимости

| Тип данных       | Accuracy | Вывод                                        |
|------------------|----------|----------------------------------------------|
| Гауссовые облака | ~99–100% | Линейно разделимы - перцептрон справляется   |
| XOR              | ~43–52%  | Хуже случайности - прямая бесполезна         |
| Окружность       | ~51–53%  | Граница кривая - прямая не поможет           |

Однослойный перцептрон строит линейную границу. Для нелинейных задач
(XOR, окружности) необходимы либо **полиномиальные признаки**,
либо **многослойные нейронные сети**.
